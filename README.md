# SBM Engine: Мультирежимная Предиктивная Аналитика (Predictive Maintenance)

Этот проект представляет собой высокопроизводительную реализацию алгоритма **Similarity-Based Modeling (SBM)** на Python. Алгоритм предназначен для мониторинга состояния промышленного оборудования, прогнозирования временных рядов и раннего обнаружения деградации.

Скрипт не просто строит прогнозы, а работает как полноценный "виртуальный диагност" (APM - Asset Performance Management), вычисляя динамические уставки, индекс здоровья оборудования и формируя тепловые карты аномалий.

## 🚀 Ключевые возможности

* **Продвинутая диагностика (Predictive Maintenance):**
  * **Динамические уставки:** Автоматический расчет "коридора нормы" (например, $\pm 3\sigma$) на основе эталонных данных.
  * **Индекс здоровья (Overall Health Score):** Сводная метрика состояния всего агрегата от 0 до 100%.
  * **Тепловая карта (Deviation Heatmap):** Визуальная матрица для быстрого обнаружения каскадных сбоев по всем датчикам одновременно.
  * **Расширенная статистика:** Расчет метрик MAPE, Макс. отклонение, MBE (систематическое смещение), RMSE и $R^2$ (коэффициент детерминации).
* **Мультирежимность (Ансамбль моделей):** Автоматически разбивает данные на сегменты в зависимости от управляющего параметра (например, "Расход") и применяет к каждому свою эталонную модель.
* **JIT-компиляция (Numba):** Математическое ядро скомпилировано с помощью `@njit(parallel=True)`, что ускоряет вычисления в десятки раз за счет распараллеливания на все ядра процессора.
* **Потоковое чтение (Chunking):** Обработка огромных файлов частями без переполнения оперативной памяти (RAM).
* **"Пуленепробиваемая" предобработка:**
  * Каскадный парсер дат (поддержка множества форматов времени в одном файле).
  * Автоматическое определение кодировок (UTF-8, cp1251 и др.).
  * Универсальное чтение десятичных дробей (точки и запятые).
  * Удаление мусора и интерполяция пропусков.

## 🛠 Установка и зависимости

Для работы скрипта требуется Python 3.8+ и ряд библиотек для анализа данных и визуализации. 

Быстрая установка всех зависимостей:
```bash
pip install -r requirements.txt
⚙️ Настройка и запускВсе основные настройки производятся в начале скрипта в блоке НАСТРОЙКИ ПОЛЬЗОВАТЕЛЯ.1. Базовые параметры и аналитикаTEST_FILE: Имя CSV-файла с тестовыми данными.SIGMA_MULTIPLIER: Ширина "коридора нормы" для расчета уставок (по умолчанию 3.0 — правило трех сигм).DATE_FORMATS: Список ожидаемых форматов времени (если оставить пустым, включится умный автопарсинг).2. Настройка режимов работыСловарь MODES позволяет задать неограниченное количество режимов. Скрипт отслеживает значение параметра MODE_PARAMETER и "на лету" применяет нужную матрицу обучения.PythonMODE_PARAMETER = 'Qmh' 

MODES = {
    'Режим_Низкая_Нагрузка': {
        'file': 'train_low_mode.csv',         
        'condition': lambda x: x < 500        
    },
    'Режим_Высокая_Нагрузка': {
        'file': 'train_high_mode.csv',        
        'condition': lambda x: x >= 500       
    }
}

Bashpython sbm_engine.py
