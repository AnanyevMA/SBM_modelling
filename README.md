SBM Engine: Мультирежимное моделирование на основе подобия (Similarity-Based Modeling)
Этот проект представляет собой высокопроизводительную реализацию алгоритма Similarity-Based Modeling (SBM) на Python. Алгоритм предназначен для мониторинга состояния промышленного оборудования, прогнозирования временных рядов и раннего обнаружения аномалий (отклонений фактических значений от эталонной модели).

Скрипт оптимизирован для работы с большими объемами промышленных данных (CSV-выгрузки с логгеров и АСУ ТП), автоматически очищает "мусорные" значения и поддерживает динамическое переключение моделей в зависимости от режима работы оборудования.

🚀 Ключевые возможности
Мультирежимность: Автоматически разбивает тестовую выборку на сегменты в зависимости от управляющего параметра (например, "Расход" или "Мощность") и применяет к каждому сегменту свою матрицу обучения.

JIT-компиляция (Numba): Математическое ядро алгоритма (расчет евклидовых расстояний, весов и прогнозов) скомпилировано с помощью @njit(parallel=True), что ускоряет вычисления в десятки раз за счет распараллеливания на все ядра процессора.

Потоковое чтение данных (Chunking): Использование pandas с параметром chunksize позволяет обрабатывать огромные файлы, не переполняя оперативную память (RAM).

"Пуленепробиваемая" предобработка: Автоматически решает типичные проблемы промышленных датасетов:

Распознавание кириллицы (cp1251) и запятых вместо десятичных точек.

Принудительная конвертация текста в числа (игнорирование мусора и системных ошибок датчиков).

Удаление полностью пустых столбцов и интерполяция пропусков средними значениями.

Автоматизированная отчетность: * Генерация графиков (Actual vs Predicted + Deviation) для каждого параметра. Очистка имен файлов от запрещенных символов Windows.

Формирование сводного Excel-отчета с расчетом метрик качества (MAPE, Максимальное отклонение) и детальными листами по каждому датчику.

🛠 Установка и зависимости
Для работы скрипта требуется Python 3.8+ и ряд библиотек для анализа данных. Установите зависимости через pip:

Bash
pip install numpy pandas matplotlib scipy numba tqdm openpyxl
⚙️ Настройка и запуск
Все основные настройки производятся в начале скрипта в блоке НАСТРОЙКИ ПОЛЬЗОВАТЕЛЯ.

1. Базовые параметры
TEST_FILE: Имя CSV-файла с тестовыми данными, для которых нужно построить прогноз.

SEPARATOR: Разделитель столбцов в ваших CSV (по умолчанию ;).

LAMBDA_REG: Коэффициент регуляризации матрицы подобия (по умолчанию 1e-3). Увеличьте, если матрица получается сингулярной.

2. Настройка режимов работы (Ансамбль моделей)
Ключевая особенность проекта — словарь MODES. Вы можете задать неограниченное количество режимов. Скрипт будет отслеживать значение параметра MODE_PARAMETER и "на лету" подтягивать нужную обучающую выборку.

Python
# Название столбца, по которому определяется режим (например, 'Qmh')
MODE_PARAMETER = 'Qmh' 

MODES = {
    'Режим_Низкая_Нагрузка': {
        'file': 'train_low_mode.csv',         # Файл с эталонными данными для этого режима
        'condition': lambda x: x < 500        # Логическое условие срабатывания
    },
    'Режим_Высокая_Нагрузка': {
        'file': 'train_high_mode.csv',        
        'condition': lambda x: x >= 500       
    }
}
Запуск
После настройки путей к файлам просто запустите скрипт:

Bash
python sbm_engine.py
Процесс выполнения будет сопровождаться информативными прогресс-барами (tqdm) и подробным логированием в консоль.
